🔗 Relacionamentos Entre Tabelas

Relacionamentos Principais

TbCIAPPICaso → tbciappiusuario

Um caso pode conter múltiplos usuários (idosos). Relacionamento 1:N

TbCIAPPICaso → TbCIAPPIAcompanhamento

Um caso possui múltiplos acompanhamentos. Relacionamento 1:N

TbCIAPPICaso → TbEvento

Um caso registra múltiplos eventos. Relacionamento 1:N

TbCIAPPIAcompanhamento → TbOrgao

Acompanhamentos com encaminhamento referem um órgão. Relacionamento N:1

TbEvento → TbTipoEvento

Cada evento é classificado por um tipo. Relacionamento N:1

TbCIAPPIILPI → TbVisitaInst

Uma ILPI recebe múltiplas visitas. Relacionamento 1:N

TbEstado → TbMunicipio

Um estado contém múltiplos municípios. Relacionamento 1:N

Integridade Referencial

O banco de dados implementa chaves estrangeiras para manter a integridade dos dados:

  • Cascata de Atualizações: Mudanças em chaves primárias são propagadas automaticamente
  • Cascata de Exclusão: Exclusão de registros pai deleta automaticamente registros dependentes
  • Restrições NOT NULL: Campos obrigatórios garantem dados completos
  • Chaves Candidatas: Alguns campos têm restrição de unicidade para evitar duplicação
